 52  """* 
 53  * When this option is passed to the update method, any 
 54  * previously touched point is locked in (remains unchanged). 
 55  * <p> 
 56  * 
 57  * For example, if the mouse is over a certain point before 
 58  * the update, and that point moves away from the mouse 
 59  * (without the mouse moving otherwise) as a consequence of 
 60  * the update, the hover feedback remains "locked in" to the 
 61  * original point, even though the mouse is no longer on top 
 62  * of that point. 
 63  * <p> 
 64  * 
 65  * This option is useful for hover widgets that modify the 
 66  * position, size, symbol of points/curves, and do not want the 
 67  * selected point/curve (and popup hover widget) to change as 
 68  * a consequence of such changes. 
 69  * <p> 
 70  * 
 71  * <i>Note:</i> If the currently touched point or the curve 
 72  * containing it is deleted, GChart sets the touched point 
 73  * reference to <tt>None</tt>. In that case, this option and 
 74  * <tt>TOUCHED_POINT_CLEARED</tt> behave the same way. 
 75  * 
 76  * 
 77  * @see #update update 
 78  * @see #TOUCHED_POINT_CLEARED TOUCHED_POINT_CLEARED 
 79  * @see #TOUCHED_POINT_UPDATED TOUCHED_POINT_UPDATED
