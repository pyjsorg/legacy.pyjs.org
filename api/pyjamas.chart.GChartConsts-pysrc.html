580  """* 
581  * Due to a well-known bug (see, for example, <a 
582  * href="http:#www.hedgerwow.com/360/dhtml/css-ie-transparent-border.html"> 
583  * this explanation on Hedger Wang's blog</a>), though white 
584  * may not be black in IE6, transparent borders certainly are. 
585  * Besides this outright bug, different browsers define which 
586  * element's background color "shines through" a transparent border 
587  * differently. For example, in FF2, the background of the element 
588  * containing the border shines through, which makes setting the 
589  * border color to "transparent" equivalent to setting the border 
590  * color to equal the background color. In IE7, the color of 
591  * the chart's background "shines through"--which is more likely 
592  * what you intended when you set a symbol's border to transparent. 
593  * <p> 
594  * 
595  * To make it easy for you to eliminate such problems, and obtain a 
596  * consistently behaving transparent-border behavior cross-browser, 
597  * this special GChart-only "color" (recognized by all GChart border 
598  * color related methods <i>except</i> 
599  * <tt>GChart.setBorderColor</tt>) causes GChart to emulate a 
600  * transparent border by eliminating the border entirely (setting 
601  * it's width to 0) and changing the size and position of the element 
602  * so as to make it look like the border is still "taking up space". 
603  * 
604  * <p> 
605  * 
606  * 
607  * <blockquote><small> <i>Note:</i>The <tt>GChart.setBorderColor</tt> 
608  * method, which sets the color of the border around the entire 
609  * chart, does <i>not</i> support this keyword because GChart's 
610  * transparent border emulation relies on changing the size of, and 
611  * shifting the position of, the transparently bordered element. But, 
612  * the position of the GChart as a whole is determined not by GChart, 
613  * but by the enclosing page. Well-known CSS tricks, such as 
614  * described in the "hedgerwow" link above, can be used if you need a 
615  * Truely transparent border around the entire chart. Or, just fake 
616  * it by setting the border color to equal the background color 
617  * of the containing page.  </small></blockquote> 
618  * 
619  * <p> 
620  * 
621  * This differs from setting the border color to "transparent" (which 
622  * you can still do should you need the "standard non-standard" 
623  * transparent border color behavior) in subtle ways that can matter 
624  * in special cases. For example, because the element is smaller than 
625  * it is with "transparent", if you draw your symbols outside the 
626  * chart rectangle, GChart will not be able to track the mouse moves 
627  * inside the transparent region (yes, this is a fine point, but 
628  * there could be other differences I'm not aware of). In almost 
629  * every other case I can think of, though, setting the border color 
630  * to this special keyword instead of "transparent" will be the 
631  * simplest way to eliminate these inconsistent transparent border 
632  * problems from your charts. 
633  * <p> 
634  * 
635  * @see Symbol#setBorderColor setBorderColor 
636  * 
637  """
